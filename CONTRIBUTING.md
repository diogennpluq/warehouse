# Вклад в проект Складской ТехКонтроль

## Руководство для разработчиков

### Ветвление

- `main` - Production ветка
- `develop` - Ветка для разработки
- `feature/*` - Фичи
- `bugfix/*` - Исправление багов
- `hotfix/*` - Срочные исправления

### Коммиты

Используйте конвенцию:
```
<type>(<scope>): <subject>

types:
- feat: новая функция
- fix: исправление бага
- docs: документация
- style: форматирование
- refactor: рефакторинг
- test: тесты
- chore: сборка, зависимости

Пример:
feat(equipment): добавить прогноз замены
fix(repairs): исправить расчёт стоимости
```

### Код-стайл

#### Go
- Следуйте [Go Code Review Comments](https://github.com/golang/go/wiki/CodeReviewComments)
- Используйте `go fmt` и `go vet`
- Покрывайте тестами критическую логику

#### TypeScript/React
- Используйте TypeScript для всей новой разработки
- Пишите функциональные компоненты с hooks
- Следуйте принципам SOLID

### Тестирование

Перед пушем:
```bash
# Бэкенд
go test -v -race ./...

# Фронтенд
npm test -- --watchAll=false
```

### PR процесс

1. Создайте PR из вашей ветки в `develop`
2. Заполните описание изменений
3. Дождитесь ревью
4. Исправьте замечания
5. После одобрения - мерж
