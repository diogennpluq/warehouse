## Рекомендуемый план интеграции

### Этап 1: Подготовка шаблонов (Manual)

Для каждого Word-документа нужно:

1. Открыть документ в Microsoft Word
2. Заменить динамические данные на плейсхолдеры в формате `{{VariableName}}`
3. Убрать конкретные примеры "расходный материал для ПУ" и заменить на плейсхолдеры
4. Сохранить как новый шаблон с суффиксом `_template.docx`

**Пример:**
- `Заявка (расходный материал для ПУ).docx` → `application_template.docx`
- Заменить конкретное название товара на `{{ProcurementTitle}}`
- Заменить дату на `{{CurrentDate}}`
- И т.д.

### Этап 2: Установка библиотеки для Word

Добавить в `go.mod`:

```bash
go get github.com/lukasjarosch/go-docx
```

Или альтернатива:

```bash
go get github.com/nguyenthenguyen/docx
```

### Этап 3: Реализация генерации Word

Создать helper-функцию в `service/document_generator.go`:

```go
func ReplaceWordPlaceholders(templatePath string, data map[string]interface{}) ([]byte, error) {
    doc, err := docx.Open(templatePath)
    if err != nil {
        return nil, err
    }
    
    for key, value := range data {
        doc.ReplaceAll("{{"+key+"}}", fmt.Sprint(value))
    }
    
    var buf bytes.Buffer
    err = doc.Write(&buf)
    return buf.Bytes(), err
}
```

### Этап 4: Обновление заглушек

Заменить заглушки в `service/procurement.go` на реальную генерацию.

## Необходимые плейсхолдеры

### Общие для всех документов

- `{{ProcurementTitle}}` - Наименование закупки
- `{{CurrentDate}}` - Текущая дата
- `{{ResponsibleName}}` - ФИО ответственного

### Для заявки

- `{{Justification}}` - Обоснование необходимости
- `{{DeliveryAddress}}` - Адрес поставки
- `{{DeliveryTerms}}` - Сроки поставки

### Для распоряжения

- `{{CommissionMembers}}` - Список членов комиссии (таблица)

### Для технического задания

- `{{Items}}` - Таблица объектов закупки
  - `{{ItemName}}` - Наименование
  - `{{KTRUCode}}` - Код КТРУ
  - `{{OKPD2Code}}` - Код ОКПД2
  - `{{Quantity}}` - Количество
  - `{{UOM}}` - Единица измерения
  - `{{Characteristics}}` - Характеристики
- `{{WarrantyMonths}}` - Гарантия
- `{{DeliveryAddress}}` - Место поставки
- `{{DeliveryTerms}}` - Сроки поставки

### Для информации к извещению

- `{{ProcedureType}}` - Тип процедуры
- `{{IsSmpSonko}}` - Преимущества для СМП/СОНКО
- `{{NMCC}}` - НМЦК
- `{{ApplicationSecurityAmount}}` - Размер обеспечения заявки
- `{{ApplicationSecurityPercent}}` - Процент обеспечения

### Для требований к заявке

- `{{ApplicationSecurityRequired}}` - Требуется ли обеспечение
- `{{ApplicationSecurityAmount}}` - Размер обеспечения

### Для проекта контракта

- `{{NMCC}}` - Цена контракта
- `{{Items}}` - Спецификация
- `{{ContractSecurityRequired}}` - Требуется ли обеспечение контракта
- `{{ContractSecurityAmount}}` - Размер обеспечения контракта
- `{{AdvancePaymentPercent}}` - Размер аванса

## Примечания

1. **Excel шаблон НМЦК**: Можно заменить `nmcc_template.xlsx` на `Приложение № 2...xlsx` после проверки совместимости структуры
2. **Формат .doc**: Файл `Распоряжение...doc` в старом формате. Рекомендуется пересохранить в `.docx` для единообразия
3. **Федеральный закон**: Файл с законом оставить как справочный материал, не использовать в генерации
