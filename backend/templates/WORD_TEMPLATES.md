## Необходимые Word-документы

Для полноценной работы системы требуется создать следующие шаблоны Word с плейсхолдерами:

### 1. Заявка на закупку (`procurement_request.docx`)

**Плейсхолдеры:**
- `{{ProcurementTitle}}` - Название закупки
- `{{Justification}}` - Обоснование необходимости
- `{{ResponsibleName}}` - ФИО ответственного
- `{{DeliveryAddress}}` - Адрес поставки
- `{{DeliveryTerms}}` - Сроки поставки
- `{{CurrentDate}}` - Текущая дата

### 2. Распоряжение о создании комиссии (`procurement_order.docx`)

**Плейсхолдеры:**
- `{{ProcurementTitle}}` - Название закупки
- `{{CommissionMembers}}` - Список членов комиссии (перебор)
- `{{ResponsibleName}}` - Председатель комиссии
- `{{CurrentDate}}` - Дата распоряжения

### 3. Техническое задание - Приложение №1 (`tech_spec.docx`)

**Плейсхолдеры:**
- `{{ProcurementTitle}}` - Название закупки
- `{{Items}}` - Таблица объектов закупки (перебор):
  - `{{ItemName}}` - Наименование
  - `{{KTRUCode}}` - Код КТРУ
  - `{{OKPD2Code}}` - Код ОКПД2
  - `{{Quantity}}` - Количество
  - `{{UOM}}` - Единица измерения
  - `{{Characteristics}}` - Характеристики (перебор)
- `{{WarrantyMonths}}` - Гарантия в месяцах
- `{{DeliveryAddress}}` - Место поставки
- `{{DeliveryTerms}}` - Сроки поставки

### 4. Информация к извещению (`notice_info.docx`)

**Плейсхолдеры:**
- `{{ProcurementTitle}}` - Название закупки
- `{{ProcedureType}}` - Тип процедуры (Электронный аукцион / Запрос котировок)
- `{{IsSmpSonko}}` - Преимущества для СМП/СОНКО (Да/Нет)
- `{{NMCC}}` - Начальная максимальная цена контракта
- `{{ApplicationSecurityAmount}}` - Размер обеспечения заявки
- `{{ApplicationSecurityPercent}}` - Процент обеспечения заявки
- `{{DeliveryAddress}}` - Место поставки
- `{{DeliveryTerms}}` - Сроки поставки

### 5. Требования к составу заявки (`bid_requirements.docx`)

**Плейсхолдеры:**
- `{{ProcurementTitle}}` - Название закупки
- `{{ProcedureType}}` - Тип процедуры
- `{{ApplicationSecurityRequired}}` - Требуется ли обеспечение заявки
- `{{ApplicationSecurityAmount}}` - Размер обеспечения заявки (если требуется)

### 6. Проект контракта (`contract_project.docx`)

**Плейсхолдеры:**
- `{{ProcurementTitle}}` - Предмет контракта
- `{{NMCC}}` - Начальная максимальная цена
- `{{Items}}` - Спецификация (таблица объектов закупки)
- `{{DeliveryAddress}}` - Место поставки
- `{{DeliveryTerms}}` - Сроки поставки
- `{{WarrantyMonths}}` - Гарантийный срок
- `{{ContractSecurityRequired}}` - Требуется ли обеспечение контракта
- `{{ContractSecurityAmount}}` - Размер обеспечения контракта
- `{{ContractSecurityPercent}}` - Процент обеспечения контракта
- `{{AdvancePaymentPercent}}` - Размер аванса

## Как создать шаблоны

### Способ 1: Вручную в Microsoft Word

1. Создайте документ Word с нужной структурой
2. Вставьте плейсхолдеры в формате `{{VariableName}}`
3. Сохраните как `.docx` в директорию `backend/templates/`

### Способ 2: Использование библиотеки go-docx

В будущем можно автоматизировать заполнение Word-шаблонов через Go:

```bash
go get github.com/lukasjarosch/go-docx
```

Пример кода:

```go
import "github.com/lukasjarosch/go-docx"

func GenerateWordDoc(templatePath string, data map[string]interface{}) ([]byte, error) {
    doc, err := docx.Open(templatePath)
    if err != nil {
        return nil, err
    }
    
    // Замена плейсхолдеров
    for key, value := range data {
        doc.ReplaceAll("{{" + key + "}}", fmt.Sprint(value))
    }
    
    // Сохранение в буфер
    var buf bytes.Buffer
    err = doc.Write(&buf)
    return buf.Bytes(), err
}
```

## Примечания

- Все плейсхолдеры должны быть в формате `{{VariableName}}` без пробелов
- Для таблиц с переменным количеством строк (объекты закупки, характеристики) потребуется более сложная логика шаблонизации
- Шрифт: Times New Roman 12pt для основного текста
- Отступы и форматирование должны соответствовать требованиям делопроизводства
